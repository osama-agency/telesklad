# Руководство по множественным получателям Telegram уведомлений

## Обзор

Система теперь поддерживает отправку уведомлений о заказах нескольким получателям одновременно. По умолчанию настроены два получателя:
- **Вы**: 125861752
- **Закупщик**: 795960178

## Как это работает

### 1. Автоматическая отправка всем получателям
При создании нового заказа система автоматически отправляет уведомление всем настроенным получателям.

### 2. Адаптивные кнопки
- **Приватные чаты** (ID > 0): Используются WebApp кнопки (открываются внутри Telegram)
- **Групповые чаты** (ID < 0): Используются обычные URL кнопки (открываются в браузере)

### 3. Детальная отчетность
Система возвращает подробную информацию о результатах отправки каждому получателю.

## API Endpoints

### Тестирование отправки всем получателям
```bash
curl -X POST http://localhost:3011/api/purchases/test-all-recipients
```

**Ответ:**
```json
{
  "success": true,
  "message": "Тестовый заказ отправлен всем получателям",
  "results": [
    {
      "chatId": "125861752",
      "success": true,
      "messageId": 50
    },
    {
      "chatId": "795960178", 
      "success": true,
      "messageId": 51
    }
  ]
}
```

### Создание заказа с отправкой всем
```bash
curl -X POST http://localhost:3011/api/purchases \
  -H "Content-Type: application/json" \
  -d '{
    "supplier": "Поставщик",
    "totalCost": 150.75,
    "isUrgent": false,
    "items": [
      {
        "name": "Товар 1",
        "quantity": 2,
        "price": 50.25,
        "total": 100.50
      }
    ]
  }'
```

### Тестирование WebApp кнопок в приватном чате
```bash
curl -X POST http://localhost:3011/api/purchases/test-webapp/125861752
```

## Настройка получателей

### Через переменные окружения
Можно настроить получателей через переменную `TELEGRAM_RECIPIENTS`:

```bash
export TELEGRAM_RECIPIENTS="125861752,795960178,другой_chat_id"
```

### В коде
По умолчанию настроены в файле `backend/src/services/telegramBot.ts`:

```typescript
const NOTIFICATION_RECIPIENTS = process.env.TELEGRAM_RECIPIENTS 
  ? process.env.TELEGRAM_RECIPIENTS.split(',').map(id => id.trim())
  : ['125861752', '795960178'] // Вы и закупщик
```

## Обработка ошибок

Если отправка в один из чатов не удалась, система продолжит отправку в остальные:

```json
{
  "success": true,
  "messageId": 50,
  "results": [
    {
      "chatId": "125861752",
      "success": true,
      "messageId": 50
    },
    {
      "chatId": "795960178",
      "success": false,
      "error": "Chat not found"
    }
  ]
}
```

## WebApp функциональность

Каждый получатель получает сообщение с кнопками:
- **✅ Подтвердить** / **❌ Отменить** - быстрые действия
- **✏️ Редактировать заказ** - открывает WebApp для редактирования

WebApp URL автоматически настраивается с правильными параметрами для каждого получателя.

## Тестирование

1. **Тест отправки всем**: `POST /api/purchases/test-all-recipients`
2. **Тест WebApp в приватном чате**: `POST /api/purchases/test-webapp/:chatId`
3. **Создание реального заказа**: `POST /api/purchases`

Все тесты возвращают подробную информацию о результатах отправки каждому получателю.
